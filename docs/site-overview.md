# Site Overview

## Who I Am

Eliya Kapach is a filmmaker, director, cinematographer, editor, and storyteller based in Israel, working internationally.

The work lives at the intersection of travel, nature, outdoor adventure, premium brands, and human stories.

Every project begins with emotion, story, and experience — the goal is to create films people remember.

## Creative Philosophy

Every frame should have intention. Prefer authentic moments over staged perfection.

Cinematography, sound design, pacing, and editing should work together to create emotion rather than showcase technical ability — the audience should *feel* something.

Prioritize: atmosphere, immersion, emotion, movement, storytelling, cinematic composition. Avoid visual trends that feel temporary or gimmicky.

## Website Goals

The website exists to:

1. Build trust immediately.
2. Showcase high-end filmmaking.
3. Position Eliya as a director, not just a videographer.
4. Generate high-quality inquiries.
5. Feel like an experience rather than a portfolio.

Every page should answer: *"Why should someone trust Eliya with their story?"*

## Homepage Section Flow

Hero → Featured Reel → Selected Work → Featured Clients → Why Work With Me → Projects by Category → Behind the Scenes → Testimonials → FAQ → Instagram → Contact → Final CTA

Note: this is the target flow from the original brief. The current implementation doesn't yet include a Featured Reel, a "Featured Clients" section distinct from the logo ticker, a "Why Work With Me" section, or Behind the Scenes as a homepage section.

## Portfolio Categories

- **Hotels & Resorts** — luxury hospitality, experiences, destinations.
- **Outdoor Brands** — adventure, equipment, apparel, action sports.
- **Commercials** — brand campaigns and product films.
- **Travel Films** — personal films and cinematic documentaries.

Each category has its own page. Each project has its own case study.

## Project Page Structure

Hero → Overview → Challenge → Creative Direction → My Role → Film → Gallery → Behind the Scenes → Deliverables → Related Projects → CTA

See also [project-page-questions.md](project-page-questions.md) for the questions each case study should answer.

## UX Principles

Navigation should always feel effortless. Every important page should lead naturally to "Start a Project." Remove unnecessary friction. The website should encourage exploration.
